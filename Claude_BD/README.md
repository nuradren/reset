# Claude BD — База данных для агентов-писателей сериала

## Проект: "Лаборатория Живого: Мицелий"

Эта папка содержит структурированные JSON-базы данных для работы AI-агентов над сериалом и связанными медиапродуктами.

---

## Структура файлов

| Файл | Содержание | Для кого |
|------|------------|----------|
| `00_project_meta.json` | Метаданные проекта, философия, продукты экосистемы | Все агенты |
| `01_inner_characters.json` | Внутренние архетипы: Голодный, Снабженец, Критик, Искра | Сценаристы персонажей |
| `02_emotions_map.json` | Карта эмоций с причинами, установками и инсайтами для трансформации | Сценаристы персонажей, диалогисты |
| `03_techniques.json` | Микро-интервенции и техники выхода из петли (30-120 сек) | Консультант по техникам |
| `04_story_references.json` | Библиотека 15+ произведений с петлями и выходами | Сценаристы сюжета |
| `05_diagnostics.json` | Система экспресс-диагностики (5 блоков) | Боты, интерактивные продукты |
| `06_life_spheres.json` | 12 жизненных сфер: мейнстрим vs альтернативы | Создатели мира, исследователи |
| `07_humanOS.json` | Модель психики как операционной системы | Все агенты (метафорический фреймворк) |
| `08_agents_config.json` | Конфигурация AI-агентов и их специализации | Шоураннер, координация |
| `09_analogues_practices.json` | Аналоги и прототипы практик | Консультанты, создатели форматов |

## Ключевые концепции

### Петля автопилота
```
Пустота → Суррогат → Вина → Пустота (повтор)
```

### Внутренние персонажи
- **Голодный** — сигнализирует о потребности
- **Снабженец** — несёт суррогат
- **Критик** — голос вины после срыва
- **Искра** — творческое живое начало

### Режиссёрские метафоры
- **Стоп-кадр** — пауза для осознания
- **Склейка** — точка вмешательства для нового сценария
- **Дубль** — возможность переиграть ситуацию
- **Перемонтаж** — изменение привычного сценария

## Как использовать

### Для AI-агентов
1. Загрузить релевантные JSON-файлы в контекст
2. Использовать терминологию из глоссария (`08_agents_config.json`)
3. Следовать методологии "Режиссуры состояний"

### Для разработки персонажей
1. Определить "петлю" персонажа
2. Назначить внутренних персонажей (кто доминирует?)
3. Найти "точку склейки" для арки трансформации
4. Подобрать технику для момента прорыва

### Для написания сцен
1. Использовать референсы из `04_story_references.json`
2. Включать конкретные техники из `03_techniques.json`
3. Связывать с эмоциями из `02_emotions_map.json`
