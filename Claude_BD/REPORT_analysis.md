## 3. Ключевые открытия

### 3.1 Структура сериала

**Логлайн:**
> Гениальный, но выгоревший режиссёр инсайт-театра обнаруживает, что его "клиенты" — это зеркала его собственных травм, специально подобранные анонимным Хакером для его интеграции.

**Твист сезона 1:** Все клиенты знают друг друга
**Твист сезона 2:** Все клиенты = части Алекса

### 3.2 Центральная петля

```
Пустота → Суррогат → Вина → Пустота (повтор)
```

4 суррогата Алекса:
1. Марихуана
2. Сексоголизм
3. Трудоголизм
4. Философский эскапизм

### 3.3 ТРИЗ-система психо-сюжетов

Адаптированы 10 принципов ТРИЗ:
- Дробление — большая травма → серия микропетель
- Инверсия — обнять тень вместо борьбы
- Превращение вреда в пользу — симптом = сигнал о потребности
- Самообслуживание — клиент сам находит решение

### 3.4 Кастанеда как движок

Перепросмотр (Recapitulation) = git rebase психики:
- Ящик = Sandbox / Монтажная комната
- Список = git log событий
- Дыхание = Garbage Collector (вдох: своя энергия, выдох: чужой код)

## 4. Как работает система генерации

### Алгоритм создания эпизода:

1. **Выбор петли** → 02_emotions_map.json
2. **Формулировка противоречия** → 11_triz_psychoplots.json
3. **Выбор ТРИЗ-принципа** → conflict_matrix
4. **Подбор техники** → 03_techniques.json
5. **Создание флэшбэка** → 10_protagonist_alex.json
6. **Артефакт** → файл для ARG
7. **Клиффхэнгер** → новый вопрос

### Роли агентов:

```
Interviewer → Showrunner → [Story Architect, Dialogue Writer, Scene Writer]
                    ↓
             [ТРИЗ-панель: Innovator + Conservative + Devil's Advocate]
                    ↓
                 Critic → Continuity → Финальное решение → commit
```

## 6. Рекомендации по использованию

### Для написания сцены:
1. Загрузить: 10_protagonist_alex.json + 01_inner_characters.json + 03_techniques.json
2. Определить: какая часть рулит, какая страдает
3. Применить технику из справочника
4. Проверить по unit-тестам из 11_triz_psychoplots.json

### Для планирования сезона:
1. Загрузить: 12_season1_structure.json + 11_triz_psychoplots.json
2. Следовать LOST-структуре: тайна + портреты + раскрытия
3. Каждый эпизод отвечает на вопрос и ставит новый

### Для ARG/бота:
1. После каждой серии — артефакт в репозиторий
2. Микро-квест в боте (05_diagnostics.json)
3. Активность "Хакера" через git diff
